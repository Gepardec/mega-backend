## Why

The business process requires that a project lead confirms receipt of the Leistungsnachweis from the employee, rather than the employee self-certifying. The responsibility was always intended to be a lead confirmation step; the original model placed it on the wrong actor.

## What Changes

- `LEISTUNGSNACHWEIS` tasks are no longer employee-owned; eligible actors become the project leads for that project
- The employee remains the `subjectEmployeeId` (the task is still about them) but cannot complete it
- Employees see `LEISTUNGSNACHWEIS` as readonly in their overview (`canComplete = false`)
- Project leads can complete `LEISTUNGSNACHWEIS` tasks (`canComplete = true`)
- `LEISTUNGSNACHWEIS` is no longer generated by self-service preparation
- If a billable project has no active leads, `LEISTUNGSNACHWEIS` is not created (consistent with `PROJECT_LEAD_REVIEW` and `ABRECHNUNG` guards)

## Capabilities

### New Capabilities

_(none)_

### Modified Capabilities

- `monthend-task-generation`: LEISTUNGSNACHWEIS moves from the employee-owned task group to the project-lead-owned group; billable + active-leads guard added
- `monthend-self-service-preparation`: LEISTUNGSNACHWEIS is no longer created during self-service preparation
- `monthend-task-completion`: Project leads can now complete LEISTUNGSNACHWEIS tasks
- `monthend-status-overview`: Employee sees LEISTUNGSNACHWEIS as readonly; lead sees it as actionable

## Impact

- `MonthEndTaskType`: completion policy for `LEISTUNGSNACHWEIS` changes from `INDIVIDUAL_ACTOR` to `ANY_ELIGIBLE_ACTOR`
- `MonthEndTask`: validation invariant — `LEISTUNGSNACHWEIS` moves from the employee-owned branch (subject must be in eligible set) to a lead-owned branch (subject present, eligible actors are leads)
- `MonthEndTaskPlanningService`: LEISTUNGSNACHWEIS generation moves from `planEmployeeOwnedTasks()` to `planProjectTasks()` alongside `PROJECT_LEAD_REVIEW`, scoped to billable projects
- `PrematureMonthEndPreparationService` (self-service): stops generating `LEISTUNGSNACHWEIS` tasks
- All existing tests covering `LEISTUNGSNACHWEIS` task creation, completion, and overview visibility need updating
